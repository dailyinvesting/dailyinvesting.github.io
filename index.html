打工人的0基础 Codex 手册：6 个实战场景（附可复制 Prompt）

一个打工人写给打工人的实操指南，1小时工作，7小时摸鱼。

写在前面：这篇文章能给你什么

2025 年，OpenAI 官方发布了一篇文章，题目是《Codex 在工作中的十大使用场景》。

这个举动本身就是一个信号： 一个曾经被定义为“给程序员用的 AI 编程工具”，开始正式对非技术人员喊话了。

OpenAI 列出的十大场景里，排名第一的是“首席参谋”——Codex 帮你整理消息、管理日历、追踪行动项。

这不是一个编程任务，这是知识工作者多年来一直试图用 Zapier、Notion 和各种 AI 助手去自动化但始终没做好的那类工作。

这就是本文要讲的事： 不是理论，不是观点，是打工人可以直接复制操作的真实场景，每个场景配上我测试过的 Prompt 模板，你读完可以立刻去试。

在开始之前，先说三件事

第一，Codex 是什么。

简单说：它是 OpenAI 推出的 AI Agent，基于对自然语言和源代码都做过训练的模型， 对结构化指令的理解和结构化输出的生成，比普通 AI 助手更精准——它理解逻辑、序列和数据里的规律。

对你来说，这意味着它处理“有规则、有结构、有重复性”的任务，比大多数工具更可靠。

第二，你需要什么权限。

2025 年新版 Codex 已包含在 ChatGPT Plus、Pro 和 Team 订阅中，无需额外付费。

第三，Codex 能力的两个层次。

本文分两个层次展开： 前五个场景是“帮你想、帮你写”——你提供素材，它生成内容，你来执行； 场景六是“帮你做”——Codex 直接操控电脑、在后台定时自动运行任务，你只需要审核结果。

前五个场景是基础，场景六是进阶，建议按顺序读。

场景一：周报、月报、汇报文档

每周五下午，你坐在电脑前，开始做同一件事：

  • 打开飞书或钉钉翻消息记录
  • 打开任务管理工具
  • 打开上周周报重新填
  • 检查措辞
  • 发给领导

整个过程快则一小时，慢则两小时。

格式几乎不变，结构几乎不变，需要的信息全部散落在你已经有的地方。

手动喂料版——把素材扔进去，拿回初稿

你是一位职场写作助手，擅长把零散的工作记录整理成清晰的周报。

我的工作记录如下：

【粘贴原始记录】

请按照以下结构生成一份周报初稿：
1. 本周完成事项
2. 进行中的事项
3. 下周计划
4. 需要支持或决策的事项

语气要求：
专业但不过于正式，用词简洁，避免空话套话。

进阶版——适合向上汇报

请帮我把周报改写成适合汇报给部门负责人的版本。

要求：
- 每条体现业务价值
- 结果数据放前面
- 删除空泛描述
- 结尾总结核心成果

文件直投版——直接上传文件

不想自己整理原始素材？

直接把会议纪要 PDF、任务 Excel、聊天记录 txt 文件全部上传给 Codex。

它会自己读取后生成周报。 

我上传了以下文件：

- 本周会议纪要.docx
- 任务完成情况.xlsx
- 群聊记录.txt

请读取这些文件，
生成一份结构化周报。

挂机自动版——Automations

最省力的方式： 你什么都不用做。

Codex 每周五自动读取：

  • Google Calendar
  • Notion
  • Gmail
  • Google Docs

然后自动生成周报草稿。 

每周五下午17:00自动执行：

1. 读取本周 Google Calendar
2. 读取 Notion 项目更新
3. 读取 Gmail
4. 读取工作记录文档

生成周报草稿，
发送到 Gmail 草稿箱。

场景二：Excel / 表格数据处理

你有一个 Excel 表，领导让你：

“分析一下这季度各渠道的转化率趋势。”

你知道应该用 VLOOKUP。 但你记不住公式。

于是开始百度、试错、报错。

四十分钟过去了， 你还没开始真正分析。

自然语言写公式

我的表格：

A列：产品名称
B列：点击次数
C列：成交次数

请帮我：
在 D 列计算转化率，
保留两位小数，
点击量为0时显示“无数据”。

数据清洗版

我的 Excel 数据很乱：

1. 姓名列有全角空格
2. 手机号格式不统一
3. 日期格式混乱

请告诉我：
- Excel 内置功能怎么处理
- 如果用 Python，给我完整脚本

文件直投版

直接上传 Excel 文件， 让 Codex 修改后返回新文件。 

请帮我：

1. 新增转化率列
2. 转化率低于5%的行标红
3. 底部新增合计
4. 返回 result.xlsx

Computer Use 直接操作 Excel

更进一步： Codex 可以直接看到你的 Excel 界面。 

@Computer 打开桌面上的 Q1报告.xlsx

1. 插入数据透视表
2. 按渠道汇总销售额
3. 保存文件

每步截图给我确认。

场景三：批量文件处理

很多重复工作本质都是： 结构固定、规则明确。

比如：

  • 合同提取
  • 日报汇总
  • 批量重命名
  • PDF 转 Excel

合同信息提取

读取文件夹里的 PDF 合同：

提取：
- 合同编号
- 甲方名称
- 签约日期
- 合同金额

输出为 Excel。

批量重命名

把图片统一重命名为：

YYYYMMDD_序号.jpg

规则：
- 优先从文件名提取日期
- 没有日期则读取创建时间

Computer Use 处理内网系统

很多公司的数据根本无法导出。

这时： Computer Use 是唯一解。 

@Computer 打开公司 ERP 系统，

导出：
- 渠道A
- 渠道B
- 渠道C

本周销售报表，
保存到桌面。

【填入】

【填入】

2. 达成情况

会议纪要

场景六：Codex 操控你的电脑

这是最重要的一次变化。

AI 不再只是回答问题。

而是： 开始替你执行。

Computer Use

Codex 可以：

  • 看到你的屏幕
  • 控制鼠标键盘
  • 打开应用
  • 填写表单
  • 点击按钮

真实场景

@Computer 打开 OA 系统，

按照 Excel 内容，
录入差旅报销：

- 日期
- 金额

- 项目编号

每条保存后截图确认。
